# PRD (Product Requirements Document) - ProPrompt 100

## 1. 프로젝트 개요
사용자들이 AI를 현업에서 더 효과적으로 활용할 수 있도록, 검증된 프롬프트 라이브러리와 더 나은 프롬프트를 만들어주는 최적화 도구를 제공하는 서비스입니다.

## 2. 목표 (Goals)
- 프롬프트 엔지니어링 지식이 부족한 사용자도 고품질 AI 결과를 얻게 함.
- 직무별 필수 프롬프트를 한곳에서 제공하여 업무 도달 속도를 높임.
- 프로페셔널한 디자인을 통해 신뢰할 수 있는 비즈니스 도구로서의 가치 전달.

## 3. 사용자 스토리 (User Stories)
- **기획자**: "단순한 기획 아이디어를 넣으면, AI가 전문적인 사업 계획서 초안용 프롬프트로 바꿔주길 원함."
- **마케터**: "다른 전문가들은 마케팅 프롬프트를 어떻게 쓰는지 참고하고 바로 복사해서 쓰고 싶음."
- **인사담당자**: "영어로 된 프롬프트도 공부하고 싶어서 한/영 전환이 자유로우면 좋겠음."

## 4. 기능적 요구사항 (Functional Requirements)

### 4.1 UI/UX
- **다크 테마 (Professional)**: 심층적인 업무 집중도를 위한 다크 모드 고정.
- **가독성**: Inter 폰트 기반의 큰 텍스트 레이아웃.
- **반응형**: 모바일 및 태스크탑 화면 최적화.

### 4.2 프롬프트 라이브러리
- **100선 상시 노출**: 10개 카테고리별 각 10개씩 총 100개 데이터 제공.
- **카테고리 필터링**: 사이드바를 통한 빠른 카테고리 이동.
- **검색 기능**: 실시간 키워드 검색 (직무, 카테고리명, 내용 등).
- **즉시 복사**: 원클릭 클립보드 복사 기능.

### 4.3 AI 최적화 엔진
- **Gemini API 연동**: 사용자의 Primitive Input을 분석하여 구조화된 Prompt로 변환.
- **실시간 처리**: 로딩 상태 표시 및 애니메이션 효과.
- **연동 기능**: 라이브러리의 프롬프트를 최적화 엔진으로 즉시 전송 가능.

## 5. 기술적 요구사항 (Technical Requirements)
- **TypeScript**: 타입 안정성 확보를 위한 TS 사용.
- **Gemini API**: `gemini-3-flash-preview` 모델 활용 (빠른 속도와 정확도).
- **Tailwind CSS**: 유틸리티 우선 스타일링을 통한 빠른 커스텀 디자인 구현.

## 6. 히스토리 요약 (Change History)
- **v1.1 (현재)**: 기본 API 키 설정 및 배포 준비 완료.
- **v1.0**: 핵심 기능(100선 라이브러리 + AI 최적화 엔진) 구현 완료.
- **Next Steps (예정)**: 사용자 커스텀 프롬프트 저장 기능(Firebase 연동), 프롬프트 공유 기능 추가.
